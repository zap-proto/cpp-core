Zap 0.5.1 has just been released with some bug fixes:

* On Windows, the `zap` tool would crash when it tried to generate an ID, e.g. when using `zap id` or when compiling a file that was missing the file ID, because it tried to get random bytes from `/dev/urandom`, which of course doesn't exist on Windows. Oops. Now it uses `CryptGenRandom()`.
* Declaring a generic method (with method-specific type parameters) inside a generic interface generated code that didn't compile.
* `joinPromises()` didn't work on an array of `Promise<void>`.
* Unnecessary error messages were being printed to the console when RPC clients disconnected.

Sorry about the bugs.

In other news, as you can see, the Zap web site now lives at `zap.org`. Additionally, the Github repo has been moved to the [Sandstorm.io organization](https://github.com/zap). Both moves have left behind redirects so that old links / repository references should continue to work.
